Shifting from "Element-by-Element" to "Whole-Array" computing.

Principle

1. Scalar Loop

Fetches, checks types, and processes data exactly one element at a time. Extremely high overhead.

2. Contiguous Memory

Aligns uniform data types strictly in memory, bypassing the interpreter's type-checking.

3. Hardware Vectorization

Single Instruction, Multiple Data (SIMD). The CPU registers compute entire blocks simultaneously.

NumPy: Under the Hood

Implicit Multithreading

NumPy often links to highly optimized C libraries (OpenBLAS, MKL) which automatically hijack all your CPU cores for matrix ops.


# Control this via environment variables!
export OMP_NUM_THREADS=1
export OPENBLAS_NUM_THREADS=1

Critical when deploying to cloud containers or combining with multiprocessing tools.

Temporary Memory Bloat

Compound vectorized operations create hidden intermediate arrays in memory.


# D = A * B + C 
# What actually happens:
temp1 = A * B        # Allocates memory!
D = temp1 + C        # Allocates memory!
del temp1            # Garbage collection

For giant arrays, consider in-place ops (e.g., np.add(A, B, out=A)) to save RAM.

The Scientist Stack

A library is a collection of pre-written, reusable code. It provides optimized foundational blocks (functions and classes) so you don't have to reinvent complex mathematical or data-processing algorithms from scratch.

SciPy

Advanced math: optimization, numerical integration, linear algebra, and signal processing.

Pandas

Data manipulation via DataFrames. Essential for cleaning and querying tabular data.

Matplotlib

The industry standard for 2D and 3D data visualization, plotting, and graphing.

Scikit-Learn

Machine Learning made easy. Out-of-the-box classification, regression, and clustering.

All of these high-level libraries are built natively on top of NumPy to guarantee raw speed.

Limits of Vectorization

State Dependence: Algorithms where step $N$ depends on step $N-1$ cannot be vectorized natively. You are forced back into a slow Python for loop.
Step 1
x[1] = f(x[0])
Step 2
x[2] = f(x[1])
Step 3
x[3] = f(x[2])
