Joblib

Lightweight pipelining in Python

The Distributor: Joblib

Easily convert list comprehensions into parallel processes.

Sequential (1 Core)


def process_file(filename):
    # Heavy CPU task...
    return result

results = [process_file(f) for f in file_list]

Parallel (All Cores)


from joblib import Parallel, delayed

@delayed
def process_file(filename):
    # Heavy CPU task...
    return result

parallel = Parallel(n_jobs=-1)
results = parallel(process_file(f) for f in file_list)

More advanced things

Joblib allows you to swap out the execution backend instantly.

backend="loky" (Default)


Parallel(n_jobs=-1, backend="loky")(
    delayed(func)(x) for x in data
)

Bypasses the GIL by spawning entirely new isolated Python processes (memory overhead).

Best for: Heavy CPU math, NumPy processing.

backend="threading"


Parallel(n_jobs=4, backend="threading")(
    delayed(download)(url) for url in urls
)

Uses threads within the same process (Subject to the GIL).

Best for: Network I/O, downloading files, Web APIs.

Limits of Parallelization

Amdahl's Law

Theoretical speedup is strictly limited by the portion of the task that cannot be parallelized (the sequential fraction $p$).

$$S(n) = \frac{1}{(1-p) + \frac{p}{n}}$$

Adding infinite cores ($n \to \infty$) only yields $S_{max} = \frac{1}{1-p}$.

IPC Overhead

Inter-Process Communication isn't free. Dispatching tiny tasks to isolated processes costs more time than computing them directly.

The Pickle Penalty

To pass data to a new process, Python must serialize it (Pickle) and deserialize it. Large complex objects severely bottleneck data transfer.
