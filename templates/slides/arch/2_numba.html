JIT: Advanced Parameters

Supercharge your compilation via the @njit() signature.

parallel=True

Distributes execution across CPU cores. Replace range() with prange().


@njit(parallel=True)
def calc(x):
    for i in prange(len(x)):
        x[i] *= 2

fastmath=True

Relaxes strict IEEE-754 standards for floating-point math. Speeds up triggers/logs.


@njit(fastmath=True)
def fast_sin(x):
    return np.sin(x)

nogil=True

Completely releases the Python Global Interpreter Lock for this function block.

Limits of JIT Compilers

API Compatibility

Compilers cannot process custom or mixed Python objects natively. No Pandas DataFrames, no dictionaries of mixed types. Stick strictly to NumPy arrays and primitives.

Warm-Up Overhead

The first function call takes seconds to compile via LLVM. If your script runs a small loop only once, JIT will make it slower overall (AOT compilation is an alternative).

No Side Effects

Pure machine code operates on RAM directly. You cannot connect to databases, send network requests, or reliably use `print()` statements inside a fully compiled `njit` block.
