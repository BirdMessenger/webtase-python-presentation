4. Out-of-Core Computations

Processing massive datasets gracefully without crashing.

Principle

1. Memory Wall

Native tools would try to allocate/load memory that does not fit in the RAM resulting in a crash.

2. Task Graphs

The Engine doesn't load data, it uses a Scheduler to map out operations as a graph of instructions (this is called lazy evaluation).

3. Streaming

Upon receiving a triggering signal the data is streamed in tiny chunks from Disk to RAM, saving only the intermediate outputs.

The Scale-Out Tool: Dask

NumPy (Crash)


import numpy as np

# Tries to load 100GB into RAM immediately.
# Result: Immediate MemoryError Crash.
x = np.random.random((100000, 100000))
result = x.mean()

Dask (Lazy)


import dask.array as da

# Lazily registers 100GB of data via chunks.
# Result: 0 bytes loaded into RAM.
x = da.random.random((100000, 100000), chunks=(1000, 1000))
result = x.mean().compute() # triggers computation

Dask: Distributed Clusters

Dask doesn't just stream chunks sequentially—it can process them in parallel.


from dask.distributed import Client, LocalCluster

# Spawns a background cluster using all your CPU cores & memory limits
cluster = LocalCluster(n_workers=4, threads_per_worker=2, memory_limit='2GB')
client = Client(cluster)

# Now, any Dask operation will be automatically distributed
# across the 4 workers, processing 4 chunks simultaneously!
result = x.mean().compute()
